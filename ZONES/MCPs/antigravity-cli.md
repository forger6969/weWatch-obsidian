# antigravity CLI (`agy`) — второе мнение и self-critic

> **Что:** агентный CLI от Google (Antigravity), используется как помощник и как **критик собственных решений** Claude.
> **Путь:** `/Users/saidazim/.local/bin/agy` | v1.0.14 | залогинен ✅

## Запуск

```bash
agy -p "<промпт>"        # single-shot, печатает ответ (основной режим)
agy -c "<промпт>"        # продолжить последний разговор
agy --model "<model>" -p "<промпт>"
agy --conversation <id>  # возобновить по ID
agy models               # список моделей
```

## Модели (`agy models`)

- Gemini 3.5 Flash (Low / Medium / High)
- Gemini 3.1 Pro (Low / High)
- Claude Sonnet 4.6 (Thinking)
- Claude Opus 4.6 (Thinking)
- GPT-OSS 120B (Medium)

## Как использовать (просьба Saidazim, 2026-06-30)

Применять `agy` по своему усмотрению как помощника И как **критика своих действий** —
прогонять планы/решения через него для второго мнения перед/после нетривиальной работы.
Даёт жёсткий, конкретный ревью без воды.

### Паттерн self-critique

```bash
agy -p "Ты строгий ревьюер. Вот мой план: <план>. Назови 3 самых слабых места — где сломается или замедлит работу. Кратко, по-русски."
```

**Пример результата:** разнёс decision tree WeWatch на 3 реальных проблемы —
git-паралич на lock-протоколе задач, блокирующий human-in-the-loop, ритуальный
5-фазный цикл для мелких задач.

## Когда звать agy

| Ситуация | Запрос к agy |
|----------|--------------|
| Перед сложной задачей | "Разбери мой план, найди дыры" |
| После реализации | "Ревью этого diff/решения, что упустил" |
| Архитектурное решение | "Альтернативы этому подходу + риски" |
| Застрял на баге | "Вот симптомы + что пробовал, гипотезы по root cause" |
