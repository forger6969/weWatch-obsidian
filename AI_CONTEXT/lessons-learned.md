## ❌ 2026-05-02 — in-progress.md не надёжен, daily note — источник правды

**Что случилось:** T-S067 был сделан на 30%, сессия оборвалась. Stop-hook очистил `in-progress.md` → `status: clear`. В следующей сессии Claude увидел `clear` и пропустил незакрытую задачу.

**Почему:** `in-progress.md` очищается автоматически при любом завершении сессии, даже неполном.

**Правило:** При старте сессии ВСЕГДА читать `DAILY/Saidazim/YYYY-MM-DD.md`. Найти все `🚧 Checkpoint T-XXX`. Для каждого проверить: есть ли `✅ Завершено: T-XXX` после него? Если нет → задача не закрыта.

**Добавлено в CLAUDE.md** как "4-шаговый протокол старта сессии" (2026-05-02).
