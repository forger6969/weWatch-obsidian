# 🏛 Активное использование скиллов — триггеры и порядок

**Project:** weWatch | **Date:** 2026-05-01 07:40 | **By:** Saidazim

## Decision

ЗАКОН: скиллы запускаются автоматически при наступлении триггера, без ожидания разрешения.

ПОРЯДОК ДЛЯ КАЖДОЙ ЗАДАЧИ:
1. spec-driven-implement  → ПЕРЕД написанием кода (написать SPEC)
2. root-cause-tracing     → при баге (трассировать причину, не гадать)
3. execute-judge-loop     → В ПРОЦЕССЕ (писать→tsc→проверять→фиксить)
4. self-reflection        → ПОСЛЕ кода, ДО коммита (7 шагов анти-галлюцинации)
5. critic-agent           → ПЕРЕД merge (3 судьи, средний балл ≥7)
6. auto-tests             → ПОСЛЕ Critic APPROVE (написать тесты)
7. visual-testing         → при изменении UI файлов (скриншоты)

CLI СКИЛЛЫ:
- /simplify        → после реализации — проверить качество
- /security-review → при изменении auth/JWT/middleware
- /review          → перед PR

ИСКЛЮЧЕНИЯ (без скиллов):
- Typo fix (1 символ) → только Self-Reflection step 6
- docs/Tasks.md изменение → без скиллов
- Config/env → только Self-Reflection step 6

Добавлено в CLAUDE.md раздел: АКТИВНОЕ ИСПОЛЬЗОВАНИЕ СКИЛЛОВ — ЗАКОН
