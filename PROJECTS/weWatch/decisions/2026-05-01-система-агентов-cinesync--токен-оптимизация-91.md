# 🏛 Система агентов CineSync — токен-оптимизация 91%

**Project:** weWatch | **Date:** 2026-05-01 10:19 | **By:** Saidazim

## Decision

ПРОБЛЕМА: Каждый агент получал CLAUDE.md (906 строк) + CLAUDE_BACKEND.md (511 строк) = 1417 строк на агент. При 5 агентах = 7085 строк повторяющегося контекста.

РЕШЕНИЕ: Создана система .claude/agents/ — каждый агент получает только свой файл (~100 строк).
Экономия: ~91% токенов на каждый вызов агента.

АГЕНТЫ (файлы в .claude/agents/):
- auth-agent.md           → services/auth/ (JWT, bcrypt, brute-force)
- content-agent.md        → services/content/ (video extraction, HLS, Redis cache)
- watchparty-agent.md     → services/watch-party/ (Socket.io, sync, room management)
- user-battle-notification-agent.md → services/user/ + battle/ + notification/
- admin-agent.md          → services/admin/ + apps/admin-ui/ (React dashboard)
- mobile-agent.md         → apps/mobile/ (React Native, Expo, Emirhan zone)
- web-agent.md            → apps/web/ (Next.js 14, SSR, SEO)
- shared-agent.md         → shared/ (lock protocol, всегда sequential)
- qa-agent.md             → read-only валидация (tsc, jest, forbidden patterns)
- orchestrator.md         → dispatch таблица + token optimization guide

ПРАВИЛО DISPATCH:

  shared/ = всегда sequential (lock protocol)
  IKKALASI задачи = backend сначала, потом mobile
